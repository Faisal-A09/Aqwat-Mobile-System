Control Logic

Temperature Demand→PLC Calculation→Compressor Staging→Pump Operation→Valve / Air Cooler→Feedback Check→SCADA Display→Alarm Handling
QMSM1 receives feedback from field panels and equipment, calculates cooling demand, commands compressors, pumps, valves and cooling equipment through the correct control panels, checks feedback, handles alarms and sends status to SCADA.
Example 1 — Production temperature rises
QCF sends temperature feedback to QMSM1 → QMSM1 calculates increased cooling demand → starts / loads compressor stages → commands pumps & valve stations → feedback returns to QMSM1 → SCADA shows updated status & trends.
Example 2 — Feedback missing
QMSM1 detects a command / feedback mismatch → alarm generated → SCADA displays alarm & affected equipment → operator action required.
Training & visualization only — not an approved PLC logic document. Actual wiring, I/O addresses, cable routing, MCC details, PLC logic, protection devices, communication protocols and commissioning must be verified from approved site drawings.

Field sensors send feedback to local panels. QCF panels and QMSM2 return feedback to QMSM1. QMSM1 sends commands to slave panels and controlled equipment. SCADA receives status, alarms, trends and operator-command confirmation through QMSM1 only.
